# EVA Foundation – Copilot Guardrails (Authoritative)

## Purpose
This file defines non-negotiable constraints extracted from the official
EVA Foundation Technical Design and ConOps document.

Copilot MUST NOT introduce content outside these boundaries.

---

## IN SCOPE
- EVA Chat architecture and operations
- EVA Domain Assistant architecture and operations
- Azure cloud hosting (Protected B)
- Role-Based Access Control (RBAC)
- Audit, logging, and compliance
- Incident management and recovery
- Accessibility and bilingual compliance
- Testing, validation, and defect prioritization

---

## OUT OF SCOPE (DO NOT ADD)
- End-user training materials
- Data governance policy creation
- UI redesign or future UI enhancements
- Future model upgrades
- Automation or autonomous execution
- Project deliverables outside EVA core capabilities
- Innovation Fund / Investment Fund project specifics

---

## OPERATIONAL BOUNDARIES
- EVA does NOT integrate with internal business systems
- EVA does NOT expose public APIs
- EVA does NOT retrain LLMs on user input
- EVA does NOT eliminate program responsibility for document stewardship
